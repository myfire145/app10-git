Identificar y aplicar el principio de programación estructurada

La programación estructurada es una técnica para escribir programas de manera clara. Para ello se utilizan únicamente tres estructuras: secuencia, selección e iteración.
Solamente con estas tres estructuras se pueden escribir todos los programas y aplicaciones posibles. Si bien los lenguajes de programación tienen un mayor repertorio de estructuras de control, éstas pueden ser construidas mediante las tres básicas citadas.

Reconocimieno del entorno de trabajo en DFD

¿Qué es DFD?

DFD es un software diseñado para construir y analizar algoritmos. Se pueden crear diagramas de ﬂujo de datos para la representación de algoritmos de programación estructurada a partir de las herramientas de edición. Después de haber ingresado el algoritmo representado por el diagrama, podrá ejecutarse, analizarse y depurarse en un entorno interactivo diseñado para este ﬁn.

En DFD podemos reconocer estructuras geométricas las cuales se refieren a un proceso en concreto equitativo a palabras clave en otros lenguajes o programas de testeo de código, cada variable a utilizar debe ser declarada previamente, dichas figuras se enlistan a continuación:

  • Rombo Nos sirve de comparación si se cumple una evaluación ó si no
  • Rectángulo Nos sirve tanto para declarar variables como para hacer cálculos
  • Trapecio Es un comando para que el usuario pueda escribir valores que ser[an guardados
  • Cono horizontal Sirve para imprimir (mostrar) tanto resultados como mensajes al usuario
  • Elipse Es lo que nos indica el Inicio y el Final de nuestro diagrama
  • Romboide Es la estructura de repetición (búcle o cíclo)

¿Qué es PseInt?

PSeInt es una herramienta para asistir a un estudiante en sus primeros pasos en programación. Mediante un simple e intuitivo pseudolenguaje en español, le permite centrar su atención en los conceptos fundamentales de la algoritmia computacional, minimizando las dificultades propias de un lenguaje y proporcionando un entorno de trabajo con numerosas ayudas y recursos didácticos.

El pseudocódigo es la representación narrativa de cada paso del algoritmo, esta narrativa obedece a la lengua nativa del programador, las variables ocupadas en nuestro algoritmo se escriben al momento de utilizarlas, a continuación un listado de las estructuras comparadas con DFD y otras nuevas

  • Leer es igual a el Trapecio
  • Escribir es igual al rombo horizontal
  • Proceso y FinProceso son iguales a los círculos de Inicio y Fin
  • Si, entonces, sino, FinSi es igual al rombo de selección
  • Mientras, Fin Mientras; Para, Hasta, con Paso, Fin Para; Repetir, Hasta Que son iguales al romboide
  • <- es igual al rectángulo
  • Segun, Hacer, FinSegun no tiene similar en DFD

Identificar y aplicar el concepto de variable

Es una palabra o identificador creado por el programador para asignarle valores


Los criterios para crear una variable son:

  • Tener numeros, letras ó ambos
  • Debe comenzar en letra
  • El nombre debe referenciar al valor que va a contener

Aplicar las variables de acuerdo a su uso

  • Por contenido
    • Numéricos

      • Aquellas en las cuales se almacenan valores numéricos tanto positivos (1,10,100), negativos (-1,-6,-90), flotantes(3.16,90.99)

    • Lógicos

      • Solo pueden tener 2 valores (verdadero o falso, 0 o 1, si o no) representa el resultado de una comparación entre 2 datos

    • Alfanumericos

      • Almacenan valores o información compuesta por letras, numeros y carácteres especiales

  • Por uso
    • De trabajo

      • Reciben el resultado de una expresión aritmética parcial o completa(area_triangulo, Promedio)

    • Contadores

      • Se dice que una variable es contador si cumple ésta función: X=X+c Significa que "X" obtiene su valor actual más el valor de "c" dónde "c" es una constante ya sea positivo o negativo

    • Acumuladores

      • Se dice que una variable es acumulador si cumple ésta función: X=X+b Significa que "X" obtiene su valor actual más el valor de "b" pero ésta no tiene valor fijo

Operaciones aritméticas y su jerarquía

La jerarquia de operaciones es la misma que en álgebra,corchetes, paréntesis (de adentro hacia afuera) potencias, raíces, división, multiplicación, suma y resta.

Desarrolla procesos de estructura condicional

  • Elementos de expresión condicional

    • En la expresión se debe cumplir con un valor; por lo cual tenemos a la expresión, la serie de acciones a realizar en cada opción del condicional y el cierre del mismo

  • Manipular la estructura de decisión If

    • En la condicional if existen 2 opciones; sí y no, verdadero y falso; por lo que tendremos que definir acciones a seguir, si se cumple la condición y en caso de que no se cumpliese simplemente se saltará las líneas de código dentro de if

  • Manipular la estructura de decisión If-else

    • Éste caso es similar al anterior pero con la diferencia de que aquí definiremos código si la condición se cumple y también si no lo hace; dado el caso anterior se saltaría las lineas de "sí" y cumpliría las lineas de else

  • Manipular la estructura de decisión else-if

    • Aquí veremos la combinación de los 2 anteriores pues contine un inicio en if el cual evalúa una condición, luego tenemos else if este evaluará otra condicion en caso de que la primera no se cumpla y al final tenemos a else el cual se ejecuta cuando ninguna de las condiciones anteriores fue "verdadera"

  • Manipular la estructura de decisión Switch

    • Damos un salto de condicional, pues éste es aún más complejo ya que comienza con su declaración y condición switch(a) y luego siguen los case x: tantos como desees y dentro de ellos se escriben las líneas a ejecutar si se elige uno de los case's los cuales se cierran con un break; . la opción default: se elige cuando escribimos una tecla o valor no puesto en los case's, éste se cierra igual que los casos

  • Manipular la estructura de decisión en forma de anidamiento

    • Estructura de anidamiento se refiere a tener estructuras específicas dentro de otras del mismo tipo; como meter un globo dentro de otro globo, así por ejemplo podemos meter estructuras de selección (if, if-else, switch) dentro de otras iguales, esto nos sirve para tener mas opciones y mejor precisión en cuanto a información

Desarrolla procesos de estructura repetitiva

  • Manipular la estructura de repetición while

    • Ésta estructura permite que se repitan las líneas de código que están dentro de while tantas veces mientras se cumpla una condición, cuando dicha condición se cumple el búcle termina y continúa con otras líneas, de otro modo ésta repetición es infinita. Ésta puede ejecutarse mínimo 0 veces. Se inicia con una variable generalmente son "acumuladoras" o "contadoras" para poder tener un final específico

  • Manipular la estructura de repetición do-while

    • Do-while similar a while la única diferencia es que ésta estructura se ejecuta mínimo 1 vez y que la condicón se escribe al cierre del ciclo; tanto ésta como la primera estructura sirven para disminuir código y ser más precisos, puesto que sin ellas tendriamos que repetir las mismas líneas de código varias veces y estas serían finitas, lo cual es lo contrario a lo que buscan y son las estructuras repetitivas

  • Manipular la estructura de repetición for

    • En este caso el cíclo no se inicia si la condición no se cumple esta se inicia en for(a = 0; a = 10; +2) a es la declaracion de nuestra variable contadora o acomuladora en el primer apartado (delimitado por punto y coma ;) seguido por el valor final que debe tener para su ciclado en el segundo apartado y al final de cuanto en cuanto aumentara la variable a. Ósea que se repetirá 6 veces

  • Manipular la estructura de repetición en forma de anidamiento

    • Al igual que en el apartado de anidamiento condicional a lo que se refiere es a introcir estructuras repetitivas en otras estructuras repetitivas, esto nos sirve para hacer pequeños formularios, por ejemplo.

Uso de variables tipo Array

Un array es un indicador que puede almacenar varios valores simultáneamente. Cada uno de estos valores se identifica mediante un número al cual se llama índice. Así, para acceder al primer elemento del array habría que usar el índice cero, para el segundo el índice uno, para el tercero el índice dos, y así sucesivamente. En el siguiente ejemplo lo comprenderemos mejor

En este pequeño programa hemos declarado un array y lo hemos instanciado después de haber preguntado al usuario cuántos elementos iba a tener. Utilizamos un búcle for para recoger todos los valores que hay que meter en el array. Hemos introducido los valores en el array: en la línea "nombres[i] = Console.ReadLine()" lo que hacemos es que al elemento "i" del array le asignamos lo que devuelva el método ReadLine. Como "i" tomará valores entre 0 y el número total de elementos menos uno rellenaremos el array completo. La condición del búcle, que es i < num, es decir, que si "i" es igual a "num" el búcle ya no se altera. Después tenemos otro búcle for para recorrer todo el array y escribir sus valores en la consola. En definitiva, para acceder a un elemento del array se usa la sintaxis "array[índice]".

Convertir los programas creados en archivos ejecutables

Agregaremos un proyecto de instalación.
Los proyectos de instalación sirven para crear archivos de Windows Installer (.msi), que se utilizan con el fin de distribuir la aplicación para su instalación en otro equipo. Existen dos tipos de proyectos de instalación: los proyectos de instalación estándar creados por instaladores para aplicaciones Windows en un equipo de destino, y los proyectos de instalación Web creados por instaladores para aplicaciones Web en un servidor Web.

Para crear un nuevo proyecto de instalación

  1. En el menú Archivo, elija Agregar y, después, haga clic en Nuevo proyecto.
  2. En el cuadro de diálogo Agregar nuevo proyecto resultante, en el recuadro Tipos de proyecto, abra el nodo Otros tipos de proyectos, a continuación abra Proyectos de instalación e implementación y después seleccione Instalador de Visual Studio.
  3. En el recuadro Plantillas, elija Proyecto de instalación para una instalación estándar, o Proyecto de programa de instalación web para una aplicación web.

Para agregar un proyecto de instalación existente a una solución

  1. En el menú Archivo, elija Agregar y, después, haga clic en Proyecto existente.
  2. En el cuadro de diálogo Agregar proyecto existente, busque en la ubicación del proyecto de instalación y haga clic en Abrir.
